PODSUMOWANIE KONFIGURACJI FIREBASE + CLOUDFLARE WORKER
=====================================================

CEL
----
Przygotowano backend powiadomień dla projektu WH40k-Data-Slate w taki sposób, aby:
1. zachować dotychczasowy web push,
2. dodać obsługę Android / Firebase Cloud Messaging (FCM),
3. przechowywać tokeny urządzeń Android w osobnym KV namespace,
4. wysyłać powiadomienia z Cloudflare Workera przez FCM HTTP v1.

=====================================================
3. CLOUDFLARE — NOWY KV DLA FCM
=====================================================

[1] Utworzono nowy KV namespace do przechowywania tokenów Firebase / Android:
- namespace name: FCM_TOKENS

[2] Podłączono nowy namespace do istniejącego workera jako binding:
- binding name: FCM_TOKENS
- namespace value: FCM_TOKENS

Końcowy stan KV bindings w workerze:
- SUBSCRIPTIONS_KV  -> PUSH_SUBSCRIPTIONS
- FCM_TOKENS       -> FCM_TOKENS

ZNACZENIE:
- SUBSCRIPTIONS_KV przechowuje web push subscriptions
- FCM_TOKENS przechowuje tokeny urządzeń Android / FCM

=====================================================
4. CLOUDFLARE — NOWE SEKRETY FIREBASE
=====================================================

Do workera dodano 3 nowe sekrety potrzebne do wysyłki przez FCM HTTP v1:

[1] FIREBASE_PROJECT_ID
Źródło:
- pole "project_id" z pliku service account JSON

[2] FIREBASE_CLIENT_EMAIL
Źródło:
- pole "client_email" z pliku service account JSON

[3] FIREBASE_PRIVATE_KEY
Źródło:
- pełna wartość pola "private_key" z pliku service account JSON

WAŻNE:
Do FIREBASE_PRIVATE_KEY została wklejona pełna zawartość, łącznie z:
- -----BEGIN PRIVATE KEY-----
- sekwencjami \n
- -----END PRIVATE KEY-----
- ewentualnym końcowym \n

Sekret NIE był ręcznie przerabiany przed zapisaniem.
To jest poprawne podejście.

Końcowy stan Variables and Secrets w workerze:
- ALLOWED_ORIGIN              [plaintext]
- TRIGGER_TOKEN               [secret]
- VAPID_PRIVATE_KEY           [secret]
- VAPID_PUBLIC_KEY            [secret]
- VAPID_SUBJECT               [secret]
- FIREBASE_PROJECT_ID         [secret]
- FIREBASE_CLIENT_EMAIL       [secret]
- FIREBASE_PRIVATE_KEY        [secret]

=====================================================

Oryginalny kod workera:

- korzystał z SUBSCRIPTIONS_KV,

stary worker nie był w stanie wysyłać nic przez FCM.

=====================================================
6. WORKER — CO ZOSTAŁO DODANE W NOWYM KODZIE
=====================================================

Wgrano nową wersję kodu workera, która:
- zachowuje dotychczasowy web push,
- dodaje obsługę FCM dla Androida.

NOWE LUB ROZSZERZONE ENDPOINTY:
--------------------------------

[1] GET /api/push/health
Zwraca teraz:
- ok
- hasVapid
- hasFirebase
- webSubscriptions
- fcmTokens

[2] POST /api/push/subscribe
Bez zmian funkcjonalnych:
- zapisuje klasyczne web push subscriptions do SUBSCRIPTIONS_KV

[3] POST /api/fcm/register
Nowy endpoint:
- zapisuje token FCM urządzenia do FCM_TOKENS

[4] POST /api/fcm/unregister
Nowy endpoint:
- usuwa token FCM urządzenia z FCM_TOKENS

[5] POST /api/push/trigger
Rozszerzony endpoint:
- nadal wysyła web push do SUBSCRIPTIONS_KV
- dodatkowo wysyła wiadomości FCM do tokenów zapisanych w FCM_TOKENS

=====================================================
7. JAK DZIAŁA NOWA LOGIKA WORKERA
=====================================================

WEB PUSH:
---------
- worker odczytuje subskrypcje z SUBSCRIPTIONS_KV
- tworzy JWT VAPID
- wysyła push na endpoint przeglądarkowy
- usuwa wygasłe subskrypcje z KV

FCM:
----
- worker odczytuje tokeny z FCM_TOKENS
- na podstawie FIREBASE_CLIENT_EMAIL i FIREBASE_PRIVATE_KEY tworzy JWT OAuth
- pobiera access token z Google OAuth
- wysyła request do:
  /v1/projects/{FIREBASE_PROJECT_ID}/messages:send
- usuwa nieaktualne / wygasłe tokeny FCM z KV

PAYLOAD:
--------
Worker normalizuje payload i obsługuje pola:
- title
- body
- icon
- badge
- tag
- url
- silent

Dla FCM:
- notification jest wysyłane tylko gdy silent = false
- dane trafiają też do sekcji data
- android.notification zawiera:
  - channel_id: default
  - tag
  - icon: ic_notification
  - click_action: OPEN_INFOCZYTNIK
