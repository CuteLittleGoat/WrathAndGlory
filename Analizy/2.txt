13. Firebase Storage nie ma być używany w tym projekcie.
14. Zwykły data.json nie ma być importowany do RTDB.
15. Do RTDB importowany jest wyłącznie firebase-import.json.

5. STRUKTURA DANYCH W FIREBASE REALTIME DATABASE
======================================================================

UWAGA: Starszy plan zakładał, że pod /datavault/live będą bezpośrednio klucze sheets i _meta. Po wykryciu problemu z importem to założenie jest zmienione.

Docelowo pod /datavault/live ma być zaimportowany wrapper z pliku firebase-import.json.

Struktura w Firebase po imporcie:

Realtime Database
└── datavault
    └── live
        ├── schemaVersion
        ├── createdAt
        ├── source
        └── dataJson

Wartości:

/datavault/live/schemaVersion
"datavault-firebase-import-v1"

/datavault/live/createdAt
ISO string, np. "2026-05-12T08:46:42.832Z"

/datavault/live/source
"Repozytorium.xlsx"

/datavault/live/dataJson
String zawierający pełny JSON.stringify(dataJsonObject), gdzie dataJsonObject jest obiektem w starym/obecnym formacie data.json.

Przykład firebase-import.json:

{
  "schemaVersion": "datavault-firebase-import-v1",
  "createdAt": "2026-05-12T08:46:42.832Z",
  "source": "Repozytorium.xlsx",
  "dataJson": "{\"sheets\":{\"Bestiariusz\":[]},\"_meta\":{\"sheetOrder\":[],\"columnOrder\":{}}}"
}

Po pobraniu snapshot.val() aplikacja ma wykonać:

const snapshotValue = snapshot.val();

if (
  snapshotValue &&
  snapshotValue.schemaVersion === "datavault-firebase-import-v1" &&
  typeof snapshotValue.dataJson === "string"
) {
  return JSON.parse(snapshotValue.dataJson);
}

Wynikiem ma być zwykły obiekt danych aplikacji:

{
  "sheets": {
    "Bestiariusz": [],
    "Bronie": [],
    "Pancerze": [],
    "Talenty": []
  },
  "_meta": {
    "sheetOrder": [],
    "columnOrder": {},
    "traits": {},
    "states": {}
  }
}

Dla kompatybilności można zostawić fallback:

return snapshotValue;

Fallback ma obsługiwać ewentualne testy ze starą strukturą, ale docelowo produkcyjnie oczekiwany jest wrapper datavault-firebase-import-v1.

======================================================================
6. DLACZEGO POTRZEBNY JEST WRAPPER firebase-import.json
======================================================================

Firebase Realtime Database nie pozwala, aby klucze obiektów zawierały znaki:

. $ # [ ] /

oraz nie pozwala na puste klucze.

Zwykły data.json jest poprawnym JSON-em, ale może być niepoprawny dla importu do RTDB, bo nazwy kolumn z Excela stają się kluczami JSON. Przykładowy problematyczny klucz:

"Ile celów/akcji"

Slash "/" w nazwie klucza powoduje błąd importu:

Invalid JSON: Keys cannot be empty or contain $#[]./

Problem dotyczy wyłącznie kluczy JSON, nie wartości tekstowych.

Znaczniki w wartościach, takie jak:

{{RED}}...{{/RED}}
{{B}}...{{/B}}
{{I}}...{{/I}}
{{S}}...{{/S}}

są bezpieczne i muszą pozostać nienaruszone.

Wrapper rozwiązuje problem, bo Firebase widzi tylko cztery bezpieczne klucze:

schemaVersion
createdAt
source
dataJson

Cała właściwa zawartość data.json jest stringiem w dataJson. Klucze ze środka data.json nie są dla Firebase kluczami bazy — są tekstem.

Nie stosować prostego zamieniania znaków w kluczach, np. "/" na "_".
To mogłoby zmienić nazwy kolumn widziane przez aplikację i zepsuć mapowanie danych.

10.6. Przykładowa implementacja logiczna

let firebaseApp;
let auth;
let database;
let authReadyPromise;

export async function initFirebaseDataAccess() {
  if (!firebaseApp) {
    firebaseApp = getApps().length ? getApp() : initializeApp(firebaseConfig);
    auth = getAuth(firebaseApp);
    database = getDatabase(firebaseApp);
    await setPersistence(auth, browserLocalPersistence);
  }

  if (!authReadyPromise) {
    authReadyPromise = new Promise((resolve) => {
      const unsubscribe = onAuthStateChanged(auth, (user) => {
        unsubscribe();
        resolve(user);
      });
    });
  }

  return { app: firebaseApp, auth, database };
}

export async function waitForAuthReady() {
  await initFirebaseDataAccess();
  return authReadyPromise;
}

export function getCurrentUser() {
  return auth ? auth.currentUser : null;
}

export async function loginWithGroupPassword(password) {
  await initFirebaseDataAccess();

  if (!password || !String(password).trim()) {
    throw new Error("EMPTY_PASSWORD");
  }

  return signInWithEmailAndPassword(auth, DATA_ACCESS_EMAIL, password);
}

export async function logoutDataAccess() {
  await initFirebaseDataAccess();
  await signOut(auth);
}

export async function loadDataVaultLive() {
  await initFirebaseDataAccess();

  const dataRef = ref(database, DATA_PATH);
  const snapshot = await get(dataRef);

  if (!snapshot.exists()) {
    throw new Error("DATA_NOT_FOUND");
  }

  return unwrapDataVaultPayload(snapshot.val());
}

export function unwrapDataVaultPayload(snapshotValue) {
  if (
    snapshotValue &&
    snapshotValue.schemaVersion === FIREBASE_IMPORT_SCHEMA_VERSION &&
    typeof snapshotValue.dataJson === "string"
  ) {
    try {
      return JSON.parse(snapshotValue.dataJson);
    } catch (error) {
      const wrapped = new Error("FIREBASE_IMPORT_DATAJSON_PARSE_FAILED");
      wrapped.cause = error;
      throw wrapped;
    }
  }

  // Fallback dla testów/starego formatu.
  return snapshotValue;
}

export function isAccessError(error) {
  const code = error && (error.code || error.message);
  return [
    "auth/wrong-password",
    "auth/invalid-credential",
    "auth/user-not-found",
    "auth/too-many-requests",
    "permission_denied",
    "PERMISSION_DENIED"
  ].some((item) => String(code || "").includes(item));
}

export function getReadableAccessError(error, lang = "pl") {
  const code = error && (error.code || error.message || "");

  if (String(code).includes("auth/invalid-credential") || String(code).includes("auth/wrong-password")) {
    return lang === "en" ? "Invalid access password." : "Nieprawidłowe hasło dostępu.";
  }

  if (String(code).includes("permission_denied") || String(code).includes("PERMISSION_DENIED")) {
    return lang === "en" ? "No permission to read private data." : "Brak uprawnień do odczytu prywatnych danych.";
  }

  if (String(code).includes("DATA_NOT_FOUND")) {
    return lang === "en" ? "Private data was not found in Firebase." : "Nie znaleziono prywatnych danych w Firebase.";
  }

  if (String(code).includes("FIREBASE_IMPORT_DATAJSON_PARSE_FAILED")) {
    return lang === "en"
      ? "Firebase data wrapper is invalid. Cannot parse dataJson."
      : "Wrapper danych w Firebase jest niepoprawny. Nie można sparsować dataJson.";
  }

  return lang === "en"
    ? "Could not load private data."
    : "Nie udało się załadować prywatnych danych.";
}

2. firebase-import.json

   - importować do /datavault/live,
   - zawiera dataJson jako string,
   - bezpieczny dla kluczy Firebase.

13.4. Funkcje pomocnicze

function buildFirebaseImportJson(dataJsonObject) {
  return {
    schemaVersion: "datavault-firebase-import-v1",
    createdAt: new Date().toISOString(),
    source: "Repozytorium.xlsx",
    dataJson: JSON.stringify(dataJsonObject)
  };
}

function downloadJsonFile(filename, objectToDownload) {
  const json = JSON.stringify(objectToDownload, null, 2);
  const blob = new Blob([json], { type: "application/json;charset=utf-8" });
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = filename;
  document.body.appendChild(a);
  a.click();
  a.remove();
  URL.revokeObjectURL(url);
}

function validateFirebaseImportObject(firebaseImportObject, originalData) {
  const forbidden = /[.$#[\]\/]/;

  for (const key of Object.keys(firebaseImportObject)) {
    if (!key || forbidden.test(key)) {
      throw new Error("firebase-import.json contains an invalid Firebase key: " + key);
    }
  }

  if (typeof firebaseImportObject.dataJson !== "string") {
    throw new Error("firebase-import.json dataJson must be a string");
  }

  const restored = JSON.parse(firebaseImportObject.dataJson);

  if (JSON.stringify(restored) !== JSON.stringify(originalData)) {
    throw new Error("firebase-import.json round-trip validation failed");
  }
}

Uwaga:
Walidacja nie ma przeszukiwać kluczy wewnątrz dataJson, bo one są tekstem, a nie kluczami Firebase.

13.5. Użycie po wygenerowaniu danych

const dataJsonObject = data; // wynik obecnego parsera
const firebaseImportObject = buildFirebaseImportJson(dataJsonObject);
validateFirebaseImportObject(firebaseImportObject, dataJsonObject);

downloadJsonFile("data.json", dataJsonObject);
setTimeout(() => downloadJsonFile("firebase-import.json", firebaseImportObject), 150);

15. PWA I CACHE
======================================================================

Service worker nie może cache’ować prywatnego payloadu danych jako publicznego zasobu.

Należy:

1. Sprawdzić service-worker.js i inne pliki cache.
2. Usunąć z precache:
   - data.json,
   - firebase-import.json,
   - Repozytorium.xlsx,
   - ../DataVault/data.json,
   - dowolne statyczne URL-e do danych.
3. Utrzymać cache app-shell:
   - HTML,
   - CSS,
   - JS,
   - grafiki,
   - fonty, jeśli obecnie cache’owane,
   - manifest.
4. Nie utrwalać automatycznie pełnego dataJson w Cache API.
5. Preferowany wariant bezpieczeństwa:
   - PWA działa offline jako powłoka,
   - prywatne dane są pobierane po zalogowaniu,
   - pełny payload danych nie jest zapisywany jako statyczny publiczny cache.
6. Jeżeli w przyszłości zostanie dodany tryb offline dla prywatnych danych, wymaga to osobnej decyzji, ostrzeżenia w dokumentacji i świadomej akceptacji, że dane będą fizycznie zapisane lokalnie na urządzeniu.

- po pierwszym poprawnym haśle Firebase Auth może utrzymać sesję lokalnie,
- ponowne otwarcie PWA nie powinno pytać o hasło, o ile sesja nadal działa,
- czyszczenie danych aplikacji / wylogowanie usuwa dostęp.

24. PRZYKŁADOWY FAŁSZYWY FIXTURE TESTOWY
======================================================================

Jeżeli agent chce dodać testowy JSON do repo, wolno używać tylko fikcyjnych danych, np.:

{
  "sheets": {
    "Test": [
      {
        "Kolumna/Test": "{{RED}}TEST{{/RED}}",
        "Opis": "{{B}}Fikcyjny opis{{/B}}"
      }
    ]
  },
  "_meta": {
    "sheetOrder": ["Test"],
    "columnOrder": {
      "Test": ["Kolumna/Test", "Opis"]
    }
  }
}

Ten test powinien potwierdzić:
- zwykły data.json może mieć problematyczny klucz "Kolumna/Test",
- firebase-import.json nadal jest bezpieczny, bo problematyczny klucz jest w stringu dataJson,
- znaczniki {{RED}} i {{B}} są zachowane,
- JSON.parse(wrapper.dataJson) odtwarza dokładny obiekt.

Nie dodawać prawdziwych danych z podręczników jako testów.
